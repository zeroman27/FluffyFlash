# Vault Wist

Минимальная структура для плана, решений и учёта работ по проекту **Wist**.

## Как открыть в Obsidian

1. Установите [Obsidian](https://obsidian.md) (или из Mac App Store).
2. **Open folder as vault** → укажите папку **`ObsidianVault`** в корне репозитория **или** симлинк **`docs/vault`** (оба ведут в один vault).
3. При первом запуске Obsidian подхватит настройки из папки **`.obsidian/`** в этом каталоге (см. ниже).

**Важно:** открывайте именно корень **`ObsidianVault`**, а не подпапку внутри него — иначе создастся вложенный «левый» vault и дублируется `.obsidian`.

## Что уже настроено (`.obsidian/`)

| Файл / поведение | Смысл |
|------------------|--------|
| **`app.json`** | Новые заметки по умолчанию → папка **`00-Inbox`**; вложения → **`attachments/`**; живой предпросмотр; обновление ссылок при переименовании; wikilinks `[[...]]`. |
| **`templates.json`** | Папка шаблонов: **`Templates/`** (команда палитры: *Insert template*). |
| **`daily-notes.json`** | Ежедневные заметки складываются в **`00-Inbox`** (имя файла `YYYY-MM-DD`). |
| **`graph.json`** | Базовые опции графа (теги, сироты). |
| **`core-plugins.json`** | Включены проводник, поиск, граф, обратные ссылки, canvas, свойства, шаблоны и др. |

Папка **`workspace.json`** в `.obsidian/` по-прежнему в **`.gitignore`** — расклад окон личный, в репозиторий не попадает.

Плагин **Obsidian Sync** в конфиге может быть включён: если не пользуетесь подпиской, отключите его в *Settings → Core plugins*.

## Структура папок

| Папка | Назначение |
|-------|------------|
| `00-Inbox` | Быстрые черновики и daily notes по умолчанию |
| `10-Project-Wist` | Продукт: видение, дорожная карта, архитектура, бэклог, сессии |
| `10-Project-Wist/Decisions` | ADR: короткие записи «почему решили так» |
| `10-Project-Wist/Sessions` | Итоги рабочих сессий (дата в имени файла) |
| `20-Reference` | Выдержки, ссылки, заметки по документации |
| `Templates` | Шаблон **Session** для новых сессий |
| `attachments` | Вложения (картинки и др.) по умолчанию |

## Новая сессия работы

1. Палитра команд (**Cmd+P**) → **Templates: Insert template** → **Session**.
2. Сохраните файл как `10-Project-Wist/Sessions/YYYY-MM-DD.md` (замените дату).
3. Либо скопируйте содержимое из **`10-Project-Wist/Sessions/_template.md`** — оно совпадает с шаблоном **`Templates/Session.md`**.

## Синхронизация с разработкой

При существенных изменениях в коде обновляйте [[10-Project-Wist/Roadmap|Roadmap]], [[10-Project-Wist/Architecture|Architecture]], при необходимости — новую сессию и ADR в `Decisions/`. Ассистент в Cursor может дополнять эти заметки по запросу или после крупных шагов.
