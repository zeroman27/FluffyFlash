# Сессия 2026-04-16

## Контекст

Лаги на экране **Source** при большом каталоге UUP; разбор трассы **Instruments Time Profiler** (`TimeProfile1`–`3`): до оптимизаций — огромные `PaintShapeLayer`, затем преобладали **SwiftUI AttributeGraph** (`AG::Subgraph`), **`MistSectionCard`**, стекло/material на macOS.

## Сделано

### Списки и модель

- **`DownloadISOViewModel`:** отфильтрованный список — **`@Published displayedBuilds`** с явным `recomputeDisplayedBuilds()` вместо computed-свойства на каждую инвалидацию модели.
- **Поиск по каталогу:** debounce ~280 ms; пустая строка — немедленный пересчёт.
- Остальные фильтры — немедленный пересчёт; счётчик **`displayedBuildsGeneration`** для синхронизации подсветки строки с reconcile.
- **`loadBuilds`:** пересчёт с `skipReconcile: true`, затем прежняя логика выбора (`preferredBuild`).

### Строки и секции на Source

- Длинный блок сборок — **`MistOpenSection`** (без одного материала на всю высоту списка).
- Строки **`UUPBuildRow`** — плоские цвета темы, без Material на каждой ячейке.
- У **`LazyVStack`** убраны лишние анимации при смене выбора.

### Общий chrome (вся апка)

- **`MistSectionCard`**, **`MistHeroBackground`**, **`MistPageHeader`** (плитка иконки) — без `.thinMaterial` / `.ultraThinMaterial`; заливки `WistTheme.surfaceElevated` + градиенты.
- **`RootView`:** сайдбар, логотип, выделенная строка навигации — без Material.
- **`WorkflowBottomBar`** — без `.thinMaterial`.

Файлы по сути: `DownloadISOView.swift`, `DownloadISOViewModel.swift`, `WistChrome.swift`, `RootView.swift`, `WorkflowBottomBar.swift`.
