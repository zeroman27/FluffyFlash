# Сессия 2026-04-23

## Цель сессии

- Переход от 5-шагового wizard к **Hybrid UX**: 3 области (Home / Library / Settings), адаптивный Home (Idle / Running / Done), автоматическое определение upgrade-кандидатов для ранее записанных Fluffy-флешек, полноценный Library с историей записей, SettingsView.

## Сделано

### Навигация

- Удалён `FluffyStep` enum и связанный sidebar-wizard.
- **`WistArea`** — новая модель с 3 разделами: `home`, `library`, `settings`. Иконки — SF Symbols (`sparkles`, `books.vertical`, `gearshape`), sidebar-ряд показывает подзаголовок и опциональный badge-счётчик (для Home).
- `RootView` полностью переписан: sidebar с `WistArea`, dynamic **upgrade pill** под меню (когда есть `DriveUpgradeOffer` и пользователь не на Home), глобальный UUP-прогресс остался на дне контент-колонки.
- Удалены: `FluffyStep`, `FluffyDoneView`, `FluffyActionsSheet`, `WorkflowBottomBar`, `CreateUSBView`, `DownloadsView`, `DownloadISOView`, `WistWorkflowChrome`.

### Home (`HomeView.swift`)

- Адаптивный экран с состояниями **Idle → Running → Done**, переключается по `EndToEndMediaPipeline.phase`.
- **Build picker** через модальный `FluffyBuildPickerSheet` (filters + search + list из `DownloadISOViewModel.displayedBuilds`).
- Language / Edition — инлайн `Picker` после загрузки `UUPDetails` / `UUPEditions`.
- Drive list — многовыбор `List(selection:)` с бейджем «Fluffy» для флешек с sidecar JSON.
- **Flash CTA** `WistCTAGradientButtonStyle` с динамическим лейблом («Flash» / «Flash (N drives)») и статус-подсказкой причины блокировки.
- **Upgrade hero** — `MistSectionCard` per offer с кнопкой «Update in one click»; вызывает `writeExistingISOToDrives` если ISO найден в кеше, иначе `runFullPipeline` с предвыбранной сборкой.
- **Expert mode disclosure** — 3 кнопки (Download / Build ISO / Write USB), показывается только при включённом `fluffy.expertMode`.
- Source toggle «Download UUP + build ISO» ↔ «Use existing ISO» (файл-пикер для готовых образов).

### Upgrade detector (`WistUSBUpgradeDetector.swift`)

- `@MainActor ObservableObject`, подключается к `DiskManager` через `attach(to:)` (вызывается из `RootView.task`).
- Кеш последнего известного стабильного билда на `(arch, language, edition)` в `UserDefaults` c TTL (по умолчанию 15 мин). `UUPDumpAPI.fetchBuilds` вызывается только при устаревшем кеше.
- `DriveUpgradeOffer` содержит `drive`, `currentMeta`, `latestBuild`, `isNewer`. Home показывает только `isNewer == true`; Library/Fluffy drives — все, с бейджем «up-to-date» если билды совпадают.
- Строковое сравнение билдов `isBuildStrictlyNewer` парсит dotted-номера.
- Настраивается из Settings: включение/выключение сетевых проверок, интервал 15/60 мин, «Check now».

### Library (`LibraryView.swift`)

- `Picker.segmented` с 4 табами: **Downloads** (UUP cache), **ISOs**, **History**, **Fluffy drives**.
- Downloads/ISOs — переиспользует `WistCache.listUUPFolders()` и сканирует `.iso`-файлы; действия Reveal / Build ISO / Delete.
- History — читает `WriteHistoryStore.entries`, каждая запись с иконкой успех/ошибка, датой, сборкой, диском; кнопка «Clear history».
- Fluffy drives — списки с бейджами из `WistUSBUpgradeDetector.offers`.

### History store (`WriteHistoryStore.swift`)

- JSON под `~/Library/Application Support/FluffyFlash/write-history.json`. Max 200 записей (drop oldest). HomeView вызывает `record(...)` после каждого завершения `EndToEndMediaPipeline` (успех или провал).

### Settings (`SettingsView.swift`)

- Interface language (`WistAppLanguage`), max parallel USB writes (1…4, Stepper; читается Home'ом через `@AppStorage("fluffy.maxConcurrentWrites")`), cache path + Reveal / Recalculate / Clear, Expert mode toggle, upgrade-check enabled + 15/60 min interval, «Check now».

## Сделано

### Фон и оболочка

- Новый фон приложения (`WistSurfaceModifiers.swift`): тёмная софт-текстура в центре — светлее, по углам темнее (radial + vignette), шум/зерно поверх; усиленная виньетка для читаемости стекла.
- **`FluffyTopBar`**: заголовок меняется по шагу; «Pick a Windows» на Source вместо статического «Fluffy Flash», убран «Open in browser».
- Навигация (`RootView` / `WistChrome`): подсвечивается контур выбранной вкладки (вместо старой полоски-индикатора). Убрана полоска.
- Иконки фильтров и top bar переведены на **SF Symbols** и увеличены; мелкие PNG заменены или переразмечены.

### Source (`DownloadISOView`)

- Убран текст «1. Pick a Windows build (UUPDump) …» и кнопка «Open in browser».
- Список сборок переведён на **`LazyVStack`** (виртуализация); сортировка по **номеру билда** (desc), а не по дате выхода.
- Sticky **confirm** в `safeAreaInset` — кнопка всегда видна, независимо от длины списка.
- Активные фильтры: подсветка соответствующих меню + интерактивные **chip-ы** с индивидуальным сбросом.

### Actions modal (бывшее «Download options»)

- **Новый файл** **`FluffyActionsSheet.swift`** — кастомная модалка в общем «fluffy»-стиле. Презентуется из **`RootView`** (доступ к `diskManager`, `e2ePipeline`, `usbWriter`, `selectedUSBDeviceIds`).
- Три **radio-card** сценария + шорткаты **1/2/3**: `uupOnly` (Download UUP), `uupAndIso` (Download + build ISO), `fullFlow` (Full flow to USB).
- **Language / Edition** — кастомные fluffy-menu rows (вместо `Form(.grouped)`).
- **ISO destination** (для `uupAndIso`): кэш по умолчанию или custom папка через **`NSOpenPanel`**; `DownloadISOViewModel.convertUUPFolderToISO` принимает `destinationFolder`, итоговый ISO переносится (`relocateISOIfNeeded`).
- **USB-picker** (для `fullFlow`): список `diskManager.drives`, **Refresh**, **инлайн** erase-acknowledge checkbox (без system alert), подсказка про admin password.
- **Summary** карточка: оценка размера, свободное место, ETA-заглушка, admin-password hint для Full flow.
- **Start** адаптивный: label меняется по сценарию, после нажатия — автопереход на **Downloads** (сценарии 1/2) или **USB** (сценарий 3) + запуск соответствующей ветки.
- Тогглы: **Reveal in Finder when done** (1/2), **Eject after flashing** (3), последний выбор сценария и флаги — через `@AppStorage`.

### Глобальный прогресс + Downloads

- Минималистичный прогресс-бар внизу приложения (gradient + процент + ETA) во время активной загрузки.
- На экране **Downloads** — детальный прогресс (кольцо + полоса + ETA), ясное состояние конвертации ISO.
