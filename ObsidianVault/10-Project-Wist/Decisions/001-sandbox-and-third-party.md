# ADR 001: Sandbox, CrystalFetch и внешние утилиты

## Статус

Принято

## Контекст

- Для списка дисков и записи USB нужны привилегированные операции (`diskutil`, позже `hdiutil`, `rsync`, `wimlib`).
- Нужен надёжный загрузчик с resume и официальный источник образов — UUPDump уже используется в open-source проекте CrystalFetch.

## Решение

1. **Отключить App Sandbox** для таргета приложения (Debug/Release).
2. **Включить код CrystalFetch** под Apache-2.0: `Downloader` + UUPDump-клиент и модели; атрибуция в заголовках файлов и в `THIRD_PARTY.md`.
3. **Не копировать** исходники WinDiskWriter (GPL-3.0); использовать только идеи (split WIM, сценарии USB).
4. Вынести общий **async `Process`** в `ProcessRunner` для скриптов и бинарей из бандла.

## Последствия

- Пользователь может видеть системные запросы доступа; ошибки `diskutil` показывать в UI.
- Распространение на GitHub должно включать NOTICE по Apache для заимствованных файлов.
- При добавлении подписанного DMG/нотаризации — учесть hardened runtime и вложенные бинарники.
